### 1. MEDIUM — Pending file writes are not atomic
# Providers: Claude, Gemini, Codex
# Location: pending.ts:16 (writePending)
# Issue: writePending uses writeFileSync directly to the final .json path.
# If the daemon reads .pending/ mid-write, it could parse partial JSON,
# delete the file as "invalid," and permanently lose the action.
# Note: Claude rates this LOW (writeFileSync is synchronous, payloads are
# small), Gemini rates CRITICAL, Codex rates HIGH.
# Fix: Write to .tmp file first, then renameSync (atomic on POSIX).
DECISION: Fix
STATUS: IMPLEMENTED — writePending now writes to .tmp file first, then renameSync to .json. 298/298 tests pass.

### 2. MEDIUM — sendInitialPrompt timing (R2-8 follow-up)
# Providers: Claude, Gemini, Codex
# Location: daemon.ts:167-168, daemon.ts:178-179
# Issue: sendInitialPrompt is called immediately after spawnAgent with no
# readiness check. The claude CLI takes seconds to start. tmux PTY
# buffering makes this work in practice, but it's not guaranteed.
# Gemini rates this CRITICAL (prompt lost); Claude rates PARTIAL (works
# via PTY buffer); Codex rates HIGH (no retry path).
# Fix: Add a small delay (e.g., 300ms setTimeout) or a pane readiness
# probe before sending the initial prompt.
DECISION: Write prompt to file, pass via -p in launch-agent.sh. Eliminates race entirely.
STATUS: IMPLEMENTED — spawnAgent() now writes initial prompt to Q<num>.prompt file in locks dir. buildClaudeCommand() passes it as second arg to launch-agent.sh, which reads it and passes via `claude -p`. Removed sendInitialPrompt() calls from scanCycle(). 298/298 tests pass.

### 3. MEDIUM — daemon.sh find command may have precedence issue
# Providers: Gemini, Codex (Claude confirms as correct)
# Location: scripts/daemon.sh:24
# Issue: The find command uses `-name '*.ts' -o -name '*.tsx'` with
# `-newer dist`. Gemini says -newer compares against dir mtime (false
# positive on every startup). Codex says -o precedence could trigger
# unnecessary builds for .tsx files.
# Claude says it's correct. The actual command is:
#   find src -newer dist -name '*.ts' -o -name '*.tsx'
# Due to find's precedence rules, -o splits this into two expressions:
#   (find src -newer dist -name '*.ts') -o (find src -name '*.tsx')
# The second branch has no -newer filter, so ANY .tsx file triggers rebuild.
# Fix: Group predicates: find src \( -name '*.ts' -o -name '*.tsx' \) -newer dist
DECISION: check 'man find' in terminal, look for rules on precedence of operators.
STATUS: CONFIRMED BUG, IMPLEMENTED FIX — man find confirms AND has higher precedence than OR. The -o split the expression so .tsx files matched without -newer. Fixed by grouping: \( -name '*.ts' -o -name '*.tsx' \) -newer dist. Syntax validated.
