## What to Look For

### 1. Regression bugs from the 13 round 2 fixes
- R2-1: Does db.transaction() in addResponse() interact correctly with the daemon's scan cycle? Could the transaction hold a write lock that blocks pending queue processing?
- R2-3: blockByGroup() now iterates — if addBlocker() throws for a non-cycle reason (e.g., FK violation), does the iteration abort or continue? Should it collect errors?
- R2-6: With two separate CLI commands (respond vs user-respond), is there any shared code path that still defaults to 'agent'?
- R2-8: sendInitialPrompt() uses sendKeys() under the hood. If the agent pane hasn't fully initialized yet, the keystrokes could be lost or interpreted by the shell rather than claude CLI. Is there a readiness check?
- R2-9: Orphan cleanup kills agents for non-Active questions. What about questions that transition Active -> Deferred -> Active quickly (via autoUnblock)? Could the orphan check kill an agent that is about to become Active again in the same cycle?
- R2-13: getUnreadQnums() bulk query — does the SQL handle the case where a question has zero responses?

### 2. New issues not caught in rounds 1 or 2
Focus on:
- **Error handling in scanCycle**: If any step throws, does the cycle continue or abort entirely? Is there try/catch around individual steps?
- **Concurrent daemon instances**: What prevents two daemon.sh processes from running simultaneously? File lock? PID check?
- **Pending queue race with qr-tool**: qr-tool writes .pending files while daemon reads them. Is there an atomic write pattern (write to temp + rename)?
- **TUI state consistency**: Dashboard polls every 3 seconds but detail view also polls. If the user is in detail view and the question gets resolved by the daemon, does the TUI handle the missing question gracefully?
- **Config validation**: parseShellConfig() reads config.sh. What happens if required variables are missing? Undefined values propagated?
- **Lockfile corruption**: createLockfile() writes JSON. If the daemon crashes mid-write, the next cycle reads a partial JSON file. Is there error handling in readLockfile()?
- **Test coverage gaps**: Are there tests for the interaction between orphan cleanup and auto-unblock in the same cycle? Tests for getUnreadQnums() returning empty results? Tests for user-respond pending action processing?
- **Memory leaks in TUI**: useEffect setInterval — is the interval properly cleaned up on unmount? Both dashboard and detail components.
- **Type widening side effects**: PendingAction author field changed from literal 'agent' to 'user'|'agent'. Are all consumers of PendingAction handling both values?
- **SQL injection surface**: Any place where user-provided strings (question titles, response text, group names) are interpolated into SQL rather than parameterized?

### 3. Cross-cutting verification
- Is the QuestionStatus type used consistently everywhere? Any remaining string literals that should use the type?
- Are all db.run() calls that modify multiple rows/tables wrapped in transactions where they should be?
- Does the daemon's scan cycle order (pending -> pipeline -> spawn/reprompt -> orphan cleanup -> dump) have any ordering bugs?
- Is the dirty flag set correctly by all write operations, ensuring exportDump() fires when needed?
