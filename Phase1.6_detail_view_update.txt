Execution Order
===============

  Phase 1 -> Phase 2 -> Phase 3 -> Phase 5a -> Phase 4 -> Phase 5b -> Phase 5c -> Phase 6

  Phase 1:  Thread cleanup          (small,  no deps)
  Phase 2:  Separator classes       (small,  no deps, can parallel with Phase 1)
  Phase 3:  Layout + hint changes   (small,  depends on Phase 2)
  Phase 5a: Remove "reply to this"  (small,  no deps, quick win)
  Phase 4:  Behavior changes        (medium, depends on Phase 3)
  Phase 5b: Context-sensitive footer (large,  depends on Phase 4)
  Phase 5c: Hotkey actions + Quote  (medium, depends on Phase 5b)
  Phase 6:  Quoted visuals          (medium, depends on Phase 5c)

Rationale:
- Phase 5a is a quick deletion with no dependencies — do it early to clean
  up the codebase before adding new features.
- Phase 4b (up/down wrapping) should be done before Phase 5b because the
  context-sensitive footer depends on knowing whether InputBox or a Response
  is selected.
- Phase 6 (quoted visuals) must come last since it requires Quote to exist.

Estimated scope: Phases 1-3 + 5a are small (one session). Phase 4b is medium.
Phases 5b/5c and 6 are the bulk of the remaining work and likely need their
own sessions.
