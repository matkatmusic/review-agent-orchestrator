GATE TEST ASSESSMENT: needs improvement

Issues with the current GATE TEST RULE:

1. Inconsistent labeling. Some steps have explicit "Gate test:" blocks
   (Steps 0.1, 3.1, 12.1-12.4) while most use "Write tests FIRST:".
   The rule says "every step MUST include an explicit Gate test block"
   but most steps do not have one. The "Write tests FIRST" block serves
   the same purpose but uses different vocabulary. The rule should
   clarify that "Write tests FIRST" tests are the gate tests.

2. Manual gates are not enforceable. Steps 12.3 and 12.4 have gate
   tests that are manual checklists, not automated tests. The rule says
   "Run npx vitest run after completing each step" but manual gates
   cannot be validated this way. The rule needs an exception clause for
   manual validation steps, or the manual steps need automated proxies.

3. No test count verification. The rule does not require recording the
   expected test count per step. A step could have 6 listed tests but
   only 4 implemented, and the gate would still pass (vitest reports
   zero failures). Adding "verify N tests ran" to the gate protocol
   would catch missing tests.

4. No regression clause. The rule says "Zero failures required before
   proceeding" but does not say "including all tests from prior steps."
   A strict reading allows running only the current step's tests and
   ignoring regressions. The rule should explicitly state: "npx vitest
   run executes ALL tests, not just the current step's tests."

Proposed improvements to the GATE TEST RULE:

  a. Add: "Tests listed under 'Write tests FIRST' are the gate tests
     for that step. The 'Gate test' label is used only for
     non-implementation steps."

  b. Add: "Steps with manual-only gates (12.3, 12.4) are exempt from
     automated enforcement but must be documented with a dated sign-off
     in the step's checkbox line."

  c. Add: "After completing each step, run npx vitest run (full suite,
     all prior tests included). Verify (1) zero failures and (2) total
     test count matches cumulative expected count for all completed
     steps."

  d. Add: "Record the pass count and total test count on the step's
     completion line, e.g.: - [x] Step 3.2 (12/12 tests, 42 total)"
