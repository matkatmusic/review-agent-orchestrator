RESOLVED DISAGREEMENTS
==========================================================================

DECISION 1: Inverse styling approach
  Using Claude's single <Text inverse> pattern. It already exists in
  issue-header.tsx:37-40 where focused fields render as
  <Text inverse={focused} bold={focused}>. Consistency wins.
  No need to introduce manual backgroundColor/color management.

DECISION 2: Disabled items in focus ring
  Excluding disabled items from the Tab ring. Disabled shortcuts are
  placeholder features (Defer, Rebase, Block, Show pane) — tabbable-but-
  inert items confuse users. The focus ring only includes actionable items.

DECISION 3: Footer action dispatch
  Adding an optional action string to the Shortcut interface. DetailView
  maps action strings to existing methods. Shortcuts without an action
  (Send, Scroll) are excluded from the focus ring — they are informational
  only.

F. Replace FOOTER_LINES with dynamic computation:

    export function computeFooterLines(shortcuts: Shortcut[], columns: number): number {
        let totalWidth = 0;
        for (let i = 0; i < shortcuts.length; i++) {
            const itemWidth = 1 + shortcuts[i].key.length + 2 + shortcuts[i].label.length;
            if (i > 0) totalWidth += 2;  // marginRight gap
            totalWidth += itemWidth;
        }
        return Math.max(1, Math.ceil(totalWidth / columns));
    }

    Keep FOOTER_LINES = 1 as deprecated alias for backward compat.

CHANGE 2: detail.tsx — Unified focus ring + footer action dispatch

C. Replace cycleField() with cycleFocus():

    The ring is built dynamically:
      Issue view slots:  [null, FIELD_GROUP, FIELD_BLOCKED_BY, FIELD_BLOCKS, footer0..N-1]
      Thread view slots: [null, footer0..N-1]

    cycleFocus(direction: 1 | -1) {
      const isInThread = this.threadStack.length > 0;
      const headerSlots = isInThread ? 0 : DETAIL_FIELD_COUNT;
      const focusable = getFocusableShortcuts(ViewType.Detail, isInThread);
      const footerSlots = focusable.length;
      const totalSlots = 1 + headerSlots + footerSlots;  // 1 for null/input

      // Determine current position in ring
      let current: number;
      if (this.focusedField === null && this.focusedFooterIndex === null) {
        current = 0;  // input
      } else if (this.focusedField !== null) {
        current = 1 + this.focusedField;  // header field
      } else {
        current = 1 + headerSlots + this.focusedFooterIndex!;  // footer
      }

      // Advance
      const next = ((current + direction) % totalSlots + totalSlots) % totalSlots;

      // Map back
      if (next === 0) {
        this.focusedField = null;
        this.focusedFooterIndex = null;
      } else if (next <= headerSlots) {
        this.focusedField = next - 1;
        this.focusedFooterIndex = null;
      } else {
        this.focusedField = null;
        this.focusedFooterIndex = next - 1 - headerSlots;
      }

      this.props.onFooterFocusChange(this.focusedFooterIndex);
      this.forceUpdate();
    }

==========================================================================
EDGE CASES
==========================================================================

1. Thread enter/exit: focusedFooterIndex reset to null (avoids stale index
   pointing at wrong shortcut set)
2. View transitions: App resets focusedFooterIndex on all navigation
3. Overlay open: handleKey returns early, so Tab/Enter don't change focus
4. Terminal resize: conversationHeight recomputes dynamically each render
5. Very narrow terminal: computeFooterLines clamps to minimum 1
