TUI REWRITE REQUIREMENTS DEBATE
================================

CONTEXT:
We are performing a complete rewrite of a React/Ink-based terminal user interface (TUI) for a review agent orchestrator application. The current implementation has significant architectural issues that need to be addressed. Your task is to produce a COMPLETE, DEFINITIVE list of functional requirements for the rewrite.

The rewrite must:
1. Convert all class components to functional components with hooks
2. Follow React best practices (proper state management, no forceUpdate, no mutation)
3. Follow Ink framework best practices (useFocus, useFocusManager, useInput, proper Box/Text composition)
4. Maintain 100% functional parity with the current implementation

=== KNOWN ANTI-PATTERNS IN CURRENT CODE ===

1. Mixed class + functional components (inconsistent)
2. forceUpdate() overuse instead of proper state management
3. Mutable mock data (direct MOCK_* mutations)
4. No error boundaries
5. Manual Tab ring management instead of useFocus/useFocusManager
6. Scattered keyboard handling (no centralized dispatcher)
7. Excessive prop drilling (columns, rows, callbacks repeated everywhere)
8. No auto-save persistence
9. Hard-coded colors/labels duplicated across files
10. Input Bridge pattern needed only because class components can't use hooks

=== INK FRAMEWORK BEST PRACTICES (from official documentation) ===

Components:
- Box: Flexbox layout via Yoga (flexDirection, justifyContent, alignItems, gap, padding, margin, borders, dimensions)
- Text: Required wrapper for ALL text (color, bold, italic, underline, dim, inverse, wrap, truncate)
- Static: Persistent output above dynamic content, renders once
- Transform: Modify string output before display
- Spacer: Flexible space along major axis
- STRICT RULE: Raw text outside <Text> throws errors. <Box> inside <Text> throws errors.

Hooks:
- useInput(callback): Keyboard capture with (input, key) signature. Key has direction flags, special keys, modifiers
- useFocus({ autoFocus }): Component focus registration, returns { isFocused }
- useFocusManager(): Navigate between focusable components (focusPrevious, focusNext, focusSpecific)
- useApp(): Application lifecycle (exit)
- useStdin/useStdout: Direct stream access
- useIsScreenReaderEnabled: Accessibility detection

State Management:
- Use standard React hooks (useState, useEffect, useMemo, useCallback)
- Proper cleanup in useEffect returns
- Batch state updates
- Memoize expensive computations

Focus System:
- useFocus() auto-registers with focus manager
- Tab/Shift+Tab navigates focus
- Only one component focused at a time
- autoFocus for initial focus

Performance:
- maxFps (default 30) controls render throttle
- incrementalRendering updates only changed lines
- Memoize expensive computations
- Conditional rendering with isFocused
- Test with large datasets

Anti-patterns to avoid:
- Raw text outside <Text>
- Uncontrolled infinite re-renders
- Blocking synchronous operations
- Ignoring focus state in input handlers
- Hard-coded widths (breaks responsive layouts)
- Console.log in render path
