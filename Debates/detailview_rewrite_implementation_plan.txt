DETAIL VIEW REWRITE — IMPLEMENTATION PLAN
==========================================

This plan implements the architecture from detailview_rewrite_debate_synthesis.txt.
All three agents (Gemini, Codex, Claude) reached consensus on the core design.
User decision: class components with forceUpdate(). setState migration deferred.


GUIDING PRINCIPLES
==================

1. Thread navigation is INTERNAL to DetailView. App never knows about threads.
2. The displayed chain is always derived from the thread stack at render time.
3. No render-phase mutations. All state changes happen in event handlers.
4. Class components with forceUpdate() throughout (matching existing codebase).
5. No new files. No new abstractions. Minimal delta from current file set.

ENTER THREAD (new method):

    enterThread() {
        const messages = this.currentMessages();   // flattenToArray of current chain
        const selected = messages[this.selectedMessage];
        if (!selected || !selected.reply) return;  // nothing to thread into

        // Mark replies seen on enter
        markRepliesSeen(selected);

        // Push current position onto stack
        this.threadStack.push({
            parent: selected,
            savedSelectedIndex: this.selectedMessage,
        });

        // Move cursor to last message in new chain
        const threadMessages = flattenToArray(selected.reply);
        this.selectedMessage = Math.max(0, threadMessages.length - 1);

        // Notify App of thread state change
        this.props.onThreadStateChange({ inThread: true });

        this.forceUpdate();
    }

EXIT THREAD (new method):

    exitThread() {
        if (this.threadStack.length === 0) {
            // Not in a thread — exit the view
            this.props.onBack();
            return;
        }

        // Pop the stack
        const popped = this.threadStack.pop()!;

        // Mark replies seen on exit (fixes the existing bug)
        markRepliesSeen(popped.parent);

        // Restore cursor position
        this.selectedMessage = popped.savedSelectedIndex;

        // Notify App of thread state change
        const stillInThread = this.threadStack.length > 0;
        this.props.onThreadStateChange({ inThread: stillInThread });

        this.forceUpdate();
    }

HANDLE KEY (rewrite — key changes marked with ***):

    handleKey = (_input: string, key: Key) => {
        if (this.overlay !== OverlayType.None) return;

        // *** Esc: exit thread if in thread, otherwise exit view
        if (key.escape) {
            this.exitThread();      // exitThread handles both cases
            return;
        }

        // *** Thread enter: Ctrl+Shift+.  (i.e. Ctrl+>)
        if (key.ctrl && key.shift && _input === '.') {
            this.enterThread();
            return;
        }

        // Thread exit is handled by Esc above — no separate keybind needed.

        // Adjacent issue navigation (unchanged)
        if (key.shift && (key.leftArrow || key.rightArrow)) {
            this.navigateToAdjacentIssue(key.rightArrow ? 1 : -1);
            return;
        }

        // Tab field cycling (unchanged)
        if (key.tab) {
            this.cycleField(key.shift ? -1 : 1);
            return;
        }

        // Enter on focused field opens overlay (unchanged)
        if (key.return && this.focusedField !== null) {
            this.openOverlayForField();
            return;
        }

        // Up/down scrolling (*** uses currentMessages instead of props)
        const messages = this.currentMessages();
        if (key.upArrow) {
            if (this.selectedMessage > 0) {
                this.selectedMessage--;
                this.forceUpdate();
            }
        } else if (key.downArrow) {
            if (this.selectedMessage < messages.length - 1) {
                this.selectedMessage++;
                this.forceUpdate();
            }
        }
    };

RENDER (rewrite — key changes marked with ***):

    render() {
        // ... overlay rendering stays the same ...

        // *** Derive the displayed chain from the thread stack
        const isInThread = this.threadStack.length > 0;
        const threadParent = isInThread
            ? this.threadStack[this.threadStack.length - 1].parent
            : null;
        const displayedRoot = threadParent
            ? threadParent.reply
            : this.props.rootResponse;

        // *** Build dep strings (only used in issue mode)
        const blockedByArr = [...this.blockedBySet];
        const blockedByStr = blockedByArr.length > 0
            ? blockedByArr.map(n => `I-${n}`).join(', ')
            : '(none)';
        const blocksArr = [...this.blocksSet];
        const blocksStr = blocksArr.length > 0
            ? blocksArr.map(n => `I-${n}`).join(', ')
            : '(none)';

        return (
            <Box flexDirection="column">
                <DetailInputBridge onKey={this.handleKey} />

                {/* *** INLINE CONDITIONAL — no ThreadParentHeader component */}
                {isInThread ? (
                    <ResponseContainer
                        response={threadParent!}
                        columns={columns}
                        selected={false}
                        hasNewReplies={false}
                    />
                ) : (
                    <IssueHeader
                        inum={inum}
                        issue={issue}
                        group={group}
                        blockedByStr={blockedByStr}
                        blocksStr={blocksStr}
                        columns={columns}
                        focusedField={this.focusedField}
                    />
                )}

                {/* *** Pass displayedRoot instead of props.rootResponse */}
                <ResponseChain
                    rootResponse={displayedRoot}
                    columns={columns}
                    height={this.conversationHeight}
                    userLastViewedAt={this.props.userLastViewedAt ?? null}
                    selectedIndex={this.selectedMessage}
                />

                <InputBox
                    value={this.inputValue}
                    focused={this.focusedField === null}
                    columns={columns}
                    onChange={this.handleInputChange}
                    onSubmit={this.handleInputSubmit}
                />
            </Box>
        );
    }

IMPLEMENTATION ORDER
========================================================================

The phases have dependencies:

    A (detail.tsx) can be done first — it's the foundation.
    B (issue-header.tsx) can be done in parallel with A.
    C (views.ts, app.tsx, header.tsx, footer.tsx) depends on A.
    D (response-chain.tsx) can be done in parallel with A/B/C.

Recommended order for a single developer:

    1. Phase D — ResponseChain cleanup (smallest, no deps)
    2. Phase B — IssueHeader cleanup (small, no deps)
    3. Phase A — DetailView rewrite (largest, depends on B for IssueHeader interface)
    4. Phase C — App/Header/Footer cleanup (depends on A for DetailViewProps)

This order means each phase's outputs are available when the next phase needs them.

EXCALIDRAW SCENARIO VALIDATION
========================================================================

These scenarios from the Excalidraw diagram are all handled by the architecture:

1. LINEAR 5-MESSAGE ISSUE CONVERSATION:
   threadStack = []
   displayedRoot = props.rootResponse
   messages = flattenToArray(rootResponse) = [msg1, msg2, msg3, msg4, msg5]
   IssueHeader renders 5-line metadata block.

2. THREAD OFF MESSAGE #1 (4 REPLIES DEEP):
   User selects msg1, presses Ctrl+Shift+..
   enterThread() pushes { parent: msg1, savedSelectedIndex: 0 }.
   displayedRoot = msg1.reply
   messages = flattenToArray(msg1.reply) = [reply1, reply2, reply3, reply4]
   Header area shows msg1 as a ResponseContainer (inline conditional).
   selectedMessage = 3 (last reply).

3. NESTED THREAD (reply to reply2 "Tweak implemented"):
   While in thread from step 2, user selects reply2, presses Ctrl+Shift+..
   enterThread() pushes { parent: reply2, savedSelectedIndex: 1 }.
   threadStack now has 2 entries.
   displayedRoot = reply2.reply
   messages = flattenToArray(reply2.reply) = [nestedReply1, nestedReply2]
   Header area shows reply2 as a ResponseContainer.
   selectedMessage = 1 (last nested reply).

4. EXIT NESTED THREAD:
   User presses Esc.
   exitThread() pops { parent: reply2, savedSelectedIndex: 1 }.
   threadStack back to 1 entry.
   displayedRoot = msg1.reply (back to first-level thread).
   selectedMessage = 1 (restored to reply2).
   markRepliesSeen(reply2) called.

5. EXIT FIRST-LEVEL THREAD:
   User presses Esc again.
   exitThread() pops { parent: msg1, savedSelectedIndex: 0 }.
   threadStack empty.
   displayedRoot = props.rootResponse (back to issue view).
   selectedMessage = 0 (restored to msg1).
   markRepliesSeen(msg1) called.
   IssueHeader renders 5-line metadata block again.

6. THREAD OFF MESSAGE #5 (BUG REPORT, 1 REPLY):
   User selects msg5 (index 4), presses Ctrl+Shift+..
   enterThread() pushes { parent: msg5, savedSelectedIndex: 4 }.
   displayedRoot = msg5.reply
   messages = [bugReply]
   selectedMessage = 0.

7. THREAD INTO MESSAGE WITH NO REPLIES:
   User selects a message with no reply. Ctrl+Right.
   enterThread() checks: if (!selected.reply) return;
   Nothing happens. Correct behavior — can't thread into nothing.
