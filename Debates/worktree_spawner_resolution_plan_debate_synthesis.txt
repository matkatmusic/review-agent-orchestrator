RESOLUTION SUMMARY BY PHASE
=============================================================================

PHASE 1: BLOCKING ISSUES (6 issues — system can't function without these)
--------------------------------------------------------------------------
Issue 1: pgrep → PID file + kill -0
  - Write $$ to .worktree-daemon.pid at daemon start
  - Singleton check: [ -f PID ] && kill -0 $(cat PID)
  - IDE detection: ps -o command= -p $(pgrep -x agy) with exact path match
  - trap 'cleanup_pidfile; exit 0' EXIT SIGINT SIGTERM SIGHUP
  - Complexity: MEDIUM

Issue 4: runOnFolderOpen trust prompt
  - Pre-set task.allowAutomaticTasks in workspace settings during scaffold
  - Add v1/templates/bootstrap.sh as fallback (npm install && tmux && claude)
  - Complexity: SMALL

Issue 5: Spawn/daemon race condition
  - Write .managed-worktree marker FIRST (before scaffold)
  - Daemon skips worktrees without marker
  - Complexity: TRIVIAL (solved by ownership marker in Phase 2 Issue 3)

Issue 6: Merge detection rewrite
  - Replace bash while-read with awk one-liner
  - Add --since="7 days ago" window to limit scan
  - Store birth commit in .managed-worktree for time-bounded search
  - Add is_worktree_squash_merged() fallback (git cherry based)
  - Complexity: MEDIUM

Issue 8: isBackground + empty problemMatcher
  - Daemon task: isBackground=false, command uses nohup fork
  - Claude/tmux tasks: add proper beginsPattern/endsPattern
  - Complexity: SMALL

Issue 16: Interactive prompt hidden in automated task
  - Add --force flag to spawnWorktree.sh (skips prompt)
  - VS Code task invocation uses --force
  - Manual terminal use gets the prompt
  - Complexity: SMALL


PHASE 2: DATA SAFETY (5 issues — could cause loss of user work)
--------------------------------------------------------------------------
Issue 2: Scaffold files are tracked
  - During scaffold: append paths to .git/info/exclude (per-worktree)
  - Scaffold files become untracked locally but don't pollute .gitignore
  - Cleanup: just rm the files, no git status confusion
  - Complexity: SMALL

Issue 3: No ownership marker
  - Write .managed-worktree file during scaffold (contains birth commit, name)
  - Daemon only acts on worktrees with this file
  - Complexity: TRIVIAL

Issue 11: git worktree remove --force data loss
  - First attempt: git worktree remove (no --force)
  - Only escalate to --force after own dirty check passed
  - Re-verify branch tip matches before git branch -D
  - Complexity: SMALL

Issue 10: set -e in daemon loop
  - Never use set -e in daemon
  - Explicit error checking on every command
  - || true on legitimately-failing commands
  - Complexity: SMALL

Issue 7/12: Daemon lifecycle + cleanup guards
  - Daemon checks ALL agy processes (main + any worktree path)
  - Only exits when zero agy processes for this project
  - Cleanup always checks IDE/tmux liveness before acting
  - Complexity: MEDIUM
