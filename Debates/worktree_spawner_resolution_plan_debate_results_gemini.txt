# Worktree Spawner Rewrite — Resolution Plan

## Phase 1: Blocking Issues

**1. Process detection (Issue: pgrep false positives + self-detection)**
*   **A) Confirmed.** Using `pgrep -f "worktreeDaemon.sh"` matches the `pgrep` command itself, causing the daemon to exit immediately or preventing it from launching.
*   **B) Exact Change:** Modify the INIT section of `worktreeDaemon.sh` to use a PID file:
    ```bash
    PID_FILE="/tmp/worktreeDaemon-$(echo -n "$REPO_ROOT" | md5sum | awk '{print $1}').pid"
    if [ -f "$PID_FILE" ] && kill -0 $(cat "$PID_FILE") 2>/dev/null; then
      exit 0
    fi
    echo $$ > "$PID_FILE"
    trap 'rm -f "$PID_FILE"; exit' EXIT INT TERM HUP
    ```
*   **D) Complexity:** Small

**2. Task automation (Issue: Single compound bootstrap script, remove isBackground)**
*   **A) Confirmed.** A VS Code task with `"isBackground": true` but an empty `problemMatcher` hangs the task engine, and `dependsOn` creates race conditions for prompt trust issues.
*   **B) Exact Change:** 
    Create a single `bootstrapWorktree.sh`:
    ```bash
    #!/bin/bash
    WT_NAME="$1"
    npm install
    tmux new-session -A -d -s "$WT_NAME" 2>/dev/null || true
    claude --permission-mode plan
    ```
    Modify `v1/templates/tasks.json` to just point to it:
    ```json
    {
      "version": "2.0.0",
      "tasks": [
        {
          "label": "Bootstrap Worktree",
          "type": "shell",
          "command": "${workspaceFolder}/bootstrapWorktree.sh __NAME__",
          "runOptions": { "runOn": "folderOpen" },
          "presentation": { "reveal": "always", "panel": "new", "focus": true }
        }
      ]
    }
    ```
*   **D) Complexity:** Medium

**3. Input validation (Issue: Actually enforce [a-zA-Z0-9_-]+ regex)**
*   **A) Confirmed.** Simple space substitution does not enforce the character set rule, allowing dangerous symbols.
*   **B) Exact Change:** Add explicit regex validation in `spawnWorktree.sh` Step 1:
    ```bash
    WORKTREE_NAME="${1// /_}"
    if [[ ! "$WORKTREE_NAME" =~ ^[a-zA-Z0-9_-]+$ ]]; then
      echo "Error: Worktree name must match [a-zA-Z0-9_-]+" >&2
      exit 1
    fi
    ```
*   **D) Complexity:** Trivial


## Phase 2: Data Safety Issues

**4. Scaffold design (Issue: Use untracked marker files, not tracked repo files)**
*   **A) Confirmed.** Using tracked templates as scaffolds creates git conflicts and permanent dirty status.
*   **B) Exact Change:** In `spawnWorktree.sh` Step 4, explicitly exclude the scaffold directories using `.git/info/exclude`:
    ```bash
    mkdir -p "${WORKTREE_PATH}/.claude" "${WORKTREE_PATH}/.vscode"
    echo ".claude/settings.json" >> "${WORKTREE_PATH}/.git/info/exclude"
    echo ".vscode/tasks.json" >> "${WORKTREE_PATH}/.git/info/exclude"
    cp "v1/templates/settings.json" "${WORKTREE_PATH}/.claude/settings.json"
    sed "s/__NAME__/${WORKTREE_NAME}/g" "v1/templates/tasks.json" > "${WORKTREE_PATH}/.vscode/tasks.json"
    ```
*   **D) Complexity:** Small

**5. Ownership (Issue: Add .managed-worktree marker during spawn)**
*   **A) Confirmed.** The daemon must not arbitrarily delete manually created worktrees.
*   **B) Exact Change:** 
    In `spawnWorktree.sh` Step 4:
    ```bash
    touch "${WORKTREE_PATH}/.managed-worktree"
    ```
    In `worktreeDaemon.sh` M.3, add a guard:
    ```bash
    [ ! -f "${worktree_path}/.managed-worktree" ] && continue
    ```
*   **D) Complexity:** Trivial

**6. Cleanup guards (Issue: Check all processes dead before removing worktree)**
*   **A) Confirmed.** We must not forcibly remove a worktree if it is actively open in an IDE, as it leads to data corruption.
*   **B) Exact Change:** In `worktreeDaemon.sh` M.3.3, verify the IDE is closed before proceeding:
    ```bash
    if pgrep -f "agy.*${worktree_path}" >/dev/null; then
      log "IDE still open for ${worktree_path}, skipping cleanup"
      continue
    fi
    ```
*   **D) Complexity:** Small


## Phase 3: Reliability Improvements

**7. Merge detection (Issue: Branch-birth metadata, limit scan window, use awk, add squash fallback)**
*   **A) Confirmed.** Piping `git log` into a bash loop is prohibitively slow. It also misses squash/rebase merges where the branch tip isn't a parent.
*   **B) Exact Change:** Replace `is_worktree_merged` in `worktreeLib.sh` with an `awk` optimization and a squash merge tree hash fallback:
    ```bash
    is_worktree_merged() {
      local branch="$1" repo="${2:-${REPO_ROOT}}"
      local tip; tip=$(git -C "$repo" rev-parse "refs/heads/$branch" 2>/dev/null) || return 1
      local base; base=$(git -C "$repo" merge-base main "$branch" 2>/dev/null)
      
      # Not merged if tip hasn't advanced past the merge base
      if [ "$tip" = "$base" ]; then return 1; fi

      # Fast-path: awk for explicit merge commit detection
      local merged_via_commit
      merged_via_commit=$(git -C "$repo" log --merges --format='%H %P' --since="1 month ago" | awk -v tip="$tip" '{ for(i=3; i<=NF; i++) if($i == tip) print $1 }')
      if [ -n "$merged_via_commit" ]; then return 0; fi
      
      # Fallback: Check for squash merge via matching tree hash on main
      local tree; tree=$(git -C "$repo" rev-parse "$tip^{tree}")
      local squash_match
      squash_match=$(git -C "$repo" log main --format="%H %T" --since="1 month ago" | awk -v tree="$tree" '$2 == tree { print $1; exit }')
      if [ -n "$squash_match" ]; then return 0; fi

      return 1
    }
    ```
*   **D) Complexity:** Medium

**8. Daemon safety (Issue: No set -e, explicit error handling, re-verify before destructive ops)**
*   **A) Confirmed.** Using `set -e` inside a background daemon loop causes premature crashes on minor failures.
*   **B) Exact Change:** 
    * Remove `set -e` from `worktreeDaemon.sh`.
    * Verify branch exists and no unexpected modified files exist immediately before deleting:
    ```bash
    if is_worktree_merged "$branch" "$REPO_ROOT"; then
      if ! has_non_scaffold_changes "$worktree_path"; then
        git -C "$REPO_ROOT" worktree remove --force "$worktree_path" || { log "ERR: rm failed"; continue; }
        git -C "$REPO_ROOT" branch -D "$branch" || log "ERR: branch rm failed"
      fi
    fi
    ```
*   **D) Complexity:** Small

**9. Daemon lifecycle (Issue: Watch ALL IDE windows, not just main)**
*   **A) Confirmed.** R10 states the daemon exits when the main IDE closes, but worktree IDEs might still be open, leading to orphaned tmux sessions.
*   **B) Exact Change:** Modify M.1 in `worktreeDaemon.sh` to check for any active IDE session related to the project:
    ```bash
    if ! pgrep -f "agy.*$(basename "${REPO_ROOT}")" >/dev/null && \
       ! pgrep -f "agy.*${REPO_ROOT}/../" >/dev/null; then
      log "All IDEs closed. Exiting."
      rm -f "$PID_FILE"
      exit 0
    fi
    ```
*   **D) Complexity:** Small


## Phase 4: Nice-to-have Improvements

**10. tmux (Issue: Use new-session -A flag)**
*   **A) Confirmed.** If a tmux session gets orphaned, `tmux new-session` will fail to start on subsequent IDE opens unless `-A` is used.
*   **B) Exact Change:** Update the tmux command inside the newly created `bootstrapWorktree.sh` to use `-A` (handled in Recommendation #2 above):
    ```bash
    tmux new-session -A -d -s "$WT_NAME" 2>/dev/null || true
    ```
*   **D) Complexity:** Trivial

## Revised Implementation Sequence
1. **`worktreeLib.sh`**: Add optimized `awk` merge detection, squash fallback, regex validation, and helper functions for checking modified state.
2. **`bootstrapWorktree.sh` (NEW)**: Create the unified bootstrap script combining `npm install`, `tmux`, and `claude`.
3. **`v1/templates/tasks.json`**: Update to execute only `bootstrapWorktree.sh`.
4. **`spawnWorktree.sh`**: Implement the `.managed-worktree` marker, explicit scaffold exclusion in `.git/info/exclude`, strict input validation, and handle the creation logic. Move daemon kickoff entirely to this script.
5. **`worktreeDaemon.sh`**: Migrate to PID-file singleton detection, remove `set -e`, add process cleanup guards, and update IDE loop detection to watch all IDEs.
6. **`.vscode/tasks.json`**: Remove the flaky background daemon task from the main repo entirely.
