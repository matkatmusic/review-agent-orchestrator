## What to Look For

### 1. Regression bugs from the 16 fixes
- Fix A: Does the transaction nest correctly? db.run() inside a transaction calls this.dirty=true — does that interact badly with the outer transaction?
- Fix H: Does the \; chaining syntax work on all tmux versions? Is esc() applied correctly to both the text and the pane ID?
- Fix I: addResponse() now does two db.run() calls (INSERT response + UPDATE question tracking) — should these be wrapped in a transaction?
- Fix K: autoUnblock() now queries ALL Deferred questions. A Deferred question with zero deps will pass !isBlocked() and move to Awaiting. Is this the correct behavior for questions deferred by the pipeline (enforceBlocked)?
- Fix O: The CTE walks blocker_qnum from blocked_qnum. Verify the direction is correct — does it detect A->B->C->A?
- Fix P: Is the SQLITE_BUSY error code detection correct for better-sqlite3? Does better-sqlite3 set .code on thrown errors?

### 2. New issues not caught in round 1
Focus especially on:
- **addResponse() atomicity**: The INSERT into responses + UPDATE of tracking columns are two separate db.run() calls. What happens if the second fails?
- **qr-tool cmdStatus() missing User_Deferred**: Check if the status summary in qr-tool-commands.ts accounts for the new User_Deferred status.
- **cmdList() unsafe cast**: qr-tool-commands.ts casts string to QuestionStatus without validation (line 46). Could this pass invalid status to listByStatus?
- **blockByGroup() vs circular deps**: blockByGroup() does a bulk INSERT OR IGNORE. It bypasses the wouldCreateCycle() check. Could it create cycles?
- **TUI polling memory/performance**: 3-second setInterval re-queries the entire DB. Any risk of resource issues or stale closure bugs?
- **Pending queue ordering**: Transient-error files are kept on disk. On next cycle, they sort before newer files (good) or after (bad)?
- **markReprompted sets last_responder=NULL**: If the daemon marks reprompted, then another user response arrives before the agent responds, does needsReprompt() correctly fire again?
- **Schema migration on existing DBs**: Adding 5 new columns (4 tracking + created_from) — does CREATE TABLE IF NOT EXISTS handle existing DBs that lack these columns?

### 3. Cross-cutting concerns
- Are there any places where 'Deferred' is hardcoded that should now also handle 'User_Deferred'?
- Does the esc() function in db.ts match the one in tmux.ts? (Code duplication, but are they identical?)
- Is the hasUnreadAgentResponse() function still used correctly everywhere, or are some call sites stale after the needsReprompt() refactor?
