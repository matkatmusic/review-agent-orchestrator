PHASE 3: MOCK DATA FOUNDATION
================================================================================

Goal: Correct mock data representing all Response relationship types.
This data drives view development in subsequent phases.

Req coverage: REQ-MOCK-001 through REQ-MOCK-010, REQ-QUO-001, REQ-QUO-002

- [ ] Step 3.1: Mock Data Schema
-----------------------------
  Define mock data that exercises every relationship type:

  Issue I-1: "Simple issue" (Active)
    - 3-message main chain: user -> agent -> user
    - No threads, no quotes

  Issue I-2: "Issue with thread" (Awaiting)
    - 4-message main chain
    - Message #2 has 2 replies (a thread)
    - Thread is unresolved

  Issue I-3: "Issue with nested threads" (Blocked, blocked by I-2)
    - 3-message main chain
    - Message #1 has a thread with 3 replies
    - Reply #2 in that thread has its own sub-thread (2 replies)
    - Outer thread resolved, inner thread unresolved

  Issue I-4: "Issue with quotes" (Active)
    - 5-message main chain
    - Message #3 quotes message #1 (quoted_response_id set)
    - Message #5 quotes message #3

  Issue I-5: "Issue with continuations" (Deferred)
    - Agent sends a long multi-paragraph response
    - Split into 3 continuation nodes (is_continuation=true)
    - User replies, agent replies with another 2-continuation message

  Issue I-6: "Resolved issue" (Resolved)
    - 2-message chain, simple

  Issue I-7: "Blocked issue" (Blocked, blocked by I-1 and I-2)
    - 1 message

  Issue I-8: "Issue with unread" (Active)
    - 4 messages, user_last_viewed_at set to before message #3
    - Messages #3 and #4 are "new" (unread agent responses)
